## 三、后端架构

### 3.1 main.py 模块划分

`main.py` 虽然是单文件，但内部按 `# =====` 分隔线划分为 9 个功能区域：

| 行范围 | 区域 | 核心内容 |
|---|---|---|
| 1-123 | 导入和配置 | FastAPI 应用、CORS、路径常量 |
| 124-188 | 数据模型 | Pydantic 模型（ChatMessage、AgentCreate 等约 20 个） |
| 189-587 | 工具函数 | `load_json`/`save_json`/`load_agents`/`load_conversations` 等数据读写 |
| 588-638 | 对话管理 | 项目 CRUD（`/api/conversations`） |
| 639-1244 | 团队管理 | 团队 CRUD + 团队成员管理 + 蓝图系统 + 技能系统 |
| 1245-2580 | 元团队 API | 设计任务 + 专家管理 + 评审闭环 + 产出查看 |
| 2581-3582 | 工具系统 | `TOOL_DEFINITIONS`（20+ 工具）+ `CORE_RULES` + `ROLE_TEMPLATES` + `execute_tool` |
| 3583-5976 | LLM 调用 | `call_llm`/`call_llm_stream`/`call_llm_stream_with_tools`/`detect_hallucination` |
| 5977-7800 | API 接口 | SSE 聊天流 + 文件操作 + 多模型配置 + 前端静态托管 |

### 3.2 核心函数调用链

用户发消息到智能体回复的完整流程：

```
用户发消息
    │
    ▼
POST /api/chat/stream  (main.py:5977, chat_stream 函数)
    │
    ├─ 1. 加载项目配置（load_conversations → 找到 conv → 获取 agents）
    ├─ 2. 保存用户消息（messages.json）
    │
    ├─ 【直接模式 direct=True】
    │   └─ call_llm_stream_with_tools()  ← 只调指定智能体
    │
    └─ 【链式调度 direct=False】
        ├─ 3a. main 智能体回复
        │   └─ call_llm_stream_with_tools()
        │       ├─ get_llm_client_and_config()  ← 获取 LLM 配置
        │       ├─ client.chat.completions.create(stream=True)  ← 调用 LLM
        │       ├─ yield "token" 事件  ← 流式返回给前端
        │       │
        │       └─ 工具调用循环（最多 12 轮）：
        │           ├─ LLM 返回 tool_calls
        │           ├─ yield "tool_start" 事件
        │           ├─ execute_tool()  ← 执行工具
        │           │   ├─ list_directory / read_file / write_file ...
        │           │   ├─ create_task / submit_task_result ...
        │           │   └─ invoke_sub_agent → collect_stream_sync()  ← 子代理
        │           ├─ yield "tool_end" 事件
        │           └─ 把工具结果喂回 LLM，继续循环
        │
        ├─ 3b. detect_hallucination()  ← 幻觉检测
        │
        ├─ 3c. 检查是否 @mention 了其他智能体
        │   └─ 并行调度被 mention 的智能体
        │       └─ call_llm_stream_with_tools()  ← 子智能体回复
        │
        └─ 3d. main 智能体汇总（可选，额外一轮）
    
    yield "done" 事件  ← 流结束
```

### 3.3 工具系统

工具系统是智能体的"手脚"，定义了智能体能执行哪些操作。

| 组件 | 位置 | 说明 |
|---|---|---|
| `TOOL_DEFINITIONS` | main.py:2626 | 20+ 工具的 JSON Schema 定义（给 LLM 看的） |
| `execute_tool()` | main.py:3583 | 工具执行入口（根据 tool_name 分发） |
| `CORE_RULES` | main.py:3247 | 全局规则提示词（注入所有智能体的 system_prompt） |
| `ROLE_TEMPLATES` | main.py:3308 | 5 种角色模板（creator/reviewer/analyst/worker/coordinator） |
| `MAIN_ONLY_TOOLS` | main.py:3211 | 主智能体专属工具（create_sub_task 等） |

**工具分类**：

| 类别 | 工具 | 可用者 |
|---|---|---|
| 文件操作 | list_directory, read_file, write_file, edit_file, move_file, copy_file, create_directory, glob, grep | 所有智能体 |
| 任务管理 | create_task, submit_task_result, list_tasks, update_task_status | 所有智能体 |
| 团队管理 | list_team_members, create_team_member | 所有智能体 |
| 子代理 | invoke_sub_agent | 仅设置了 can_invoke_sub_agent=true 的智能体 |
| 主智能体专属 | create_sub_task, create_todo_list, update_todo_status | 仅 main |

### 3.4 LLM 调用层级

```
call_llm()                    ← 同步调用，返回完整字符串（用于 regenerate 等）
call_llm_stream()             ← 流式调用，yield token（无工具）
call_llm_stream_with_tools()  ← 流式 + 工具循环（核心，用于 chat/stream）
collect_stream_sync()         ← 同步包装器（用于子代理调用）
get_llm_client_and_config()   ← 获取 LLM 客户端和配置
```

**配置优先级**：智能体 `model_config.config_id` > 会话级 `default_config_id` > 全局默认 `load_api_config()`

### 3.5 元团队独立模块

元团队系统拆分为 5 个独立 Python 模块：

| 模块 | 职责 |
|---|---|
| `meta_team_config.py` | 专家管理（CRUD + 提示词历史 + 回滚） |
| `meta_team_engine.py` | 三阶段执行引擎（方案撰写 → 评审 → 融合） |
| `meta_team_task.py` | 设计任务管理 + 产出文件存储 |
| `meta_team_template.py` | 专家系统提示词模板 |
| `meta_team_review.py` | 项目评审闭环（采集数据 → 专家诊断 → 报告） |

## 七、核心机制速查

### 7.1 智能体三级配置

```
全局模板 (data.json)
    └─ main 智能体（项目经理，全权限）

团队级 (teams.json)
    └─ 团队创建时从全局模板复制 main，可添加团队成员

项目级 (conversations.json)
    └─ 项目创建时从团队复制全部智能体，可独立编辑
```

修改全局模板不影响已有项目；修改团队配置不影响已有项目。

### 7.2 调度模式

| 模式 | 触发条件 | 说明 |
|---|---|---|
| 直接模式 | `direct: true` | 只调用指定智能体，不触发链式调度 |
| 链式调度 | `direct: false`（默认） | main 回复 → 检查 @mention → 调度被 mention 的智能体 → main 汇总 |
| 并行调度 | 多个智能体被 @mention | 最多 5 个并行（`MAX_PARALLEL=5`） |
| 子代理调用 | `invoke_sub_agent` 工具 | 临时创建子代理执行子任务，非递归（子代理不能再调子代理） |

### 7.3 防幻觉四层防御

| 层 | 机制 | 位置 |
|---|---|---|
| 1 | 提示词硬约束 | `CORE_RULES` 注入所有智能体 |
| 2 | 写入回读校验 | `write_file`/`edit_file`/`move_file` 执行后回读验证 |
| 3 | 任务提交校验 | `submit_task_result` 校验 task_id + 归属 + 状态 + 文件真实性 |
| 4 | 调度循环幻觉检测 | `detect_hallucination()` 扫描回复中的 task_id 引用 |

### 7.4 权限隔离

| 目录 | main | 子智能体 | 子代理 |
|---|---|---|---|
| `.agent/tasks/{自己的task}/` | 全部 | 读写自己的 | 继承父智能体 |
| `agent_work/{自己的id}/` | 全部 | 读写自己的 | 继承父智能体 |
| `shared/` | 读写 | 读写 | 读写 |
| `deliverables/` | 读写 | 只读 | 只读 |
| `.agent/tasks/{别人的task}/` | 读写 | 拒绝 | 拒绝 |

## 八、开发环境搭建

### 8.1 依赖安装

```powershell
# 后端
cd backend
pip install -r requirements.txt
# 依赖：fastapi, uvicorn, openai, pydantic, python-multipart

# 前端
cd frontend
npm install
```

### 8.2 LLM 配置

1. 启动后端后访问 `http://localhost:8000`
2. 点击右上角 ⚙ 设置 → 模型配置
3. 添加配置：
   - **base_url**：API 地址（如 `https://dashscope.aliyuncs.com/compatible-mode/v1`）
   - **api_key**：API 密钥
   - **model**：模型名（如 `deepseek-v4-flash`）
4. 保存后设为默认配置

### 8.3 启动

```powershell
# 方式 1：后端 + 前端开发模式（推荐开发时用）
# 终端 1：后端
cd backend
python main.py
# → 监听 http://localhost:8000

# 终端 2：前端（热更新）
cd frontend
npm run dev
# → 监听 http://localhost:5173，自动代理到后端

# 方式 2：后端托管前端构建产物（部署用）
cd frontend
npm run build
cd ../backend
python main.py
# → 访问 http://localhost:8000
```

### 8.4 数据文件位置

首次启动时，后端自动在 `backend/data/` 下创建所有 JSON 文件（如果不存在）。无需手动初始化。

4. 如需调用后端，在 `api/index.js` 加方法

### 9.4 新增数据文件

1. 在 `backend/data/` 下创建 JSON 文件
2. 在 `main.py` 加 `load_xxx()` / `save_xxx()` 函数（参考 `load_agents` 模式）
3. 使用 `load_json(path, default)` / `save_json(path, data)` 读写
4. 在 `AGENTS.md` 第十节登记

## 十、维护注意事项

| 事项 | 说明 |
|---|---|
| **不要拆分 main.py** | 原型阶段单文件便于快速迭代，拆分会增加导航成本 |
| **前后端字段同步** | 改后端返回格式时必须同步改前端解析 |
| **通义千问容错** | `function.arguments` 四层容错解析不要移除 |
| **API Key 脱敏** | 后端返回脱敏值，前端编辑时检测 `***` 要删除字段 |
| **数组更新用 splice** | Vue3 直接索引赋值不触发响应式 |
| **每次只改一个点** | 验证后再改下一个 |
| **先读再改** | 用函数名搜索定位，不依赖行号 |
