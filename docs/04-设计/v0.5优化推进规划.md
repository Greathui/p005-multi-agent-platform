## 三、技术设计要点

### P0-1 智能体记忆系统

**数据结构**：
```python
# agent 对象新增字段
"memory_log": [
    {
        "id": "mem_xxx",
        "type": "preference|lesson|pattern|fact",
        "content": "用户偏好Markdown格式报告",
        "source_task_id": "task_xxx",  # 来源任务
        "created_at": "2026-07-22T..."
    }
]
```

**记忆注入点**：在 `build_conversation_history` 中，将 memory_log 追加到 system_prompt 末尾。

**记忆提取**：在 `submit_task_result` 执行成功后，异步调用 LLM 从对话中提取关键经验。

**API 端点**：
- `GET /api/conversations/{conv_id}/agents/{agent_id}/memory` — 获取记忆
- `DELETE /api/conversations/{conv_id}/agents/{agent_id}/memory` — 清空记忆

### P0-2 跨项目经验迁移

**数据结构**：
```python
# teams.json agent 新增字段
"shared_memory": [
    {"type": "...", "content": "...", "source_project": "proj_xxx", "created_at": "..."}
]
```

**同步机制**：项目级 memory_log 超过 10 条时，自动将高价值记忆上传到团队级 shared_memory。

**继承机制**：创建新项目时，将团队级 shared_memory 复制到项目级 memory_log。

### P0-3 对话摘要压缩

**触发条件**：当 `build_conversation_history` 中消息 token 数超过截断阈值时。

**摘要结构**：
```python
{
    "key_decisions": ["决策1", "决策2"],
    "role_settings": ["角色设定1"],
    "completed_tasks": ["已完成任务1"],
    "user_preferences": ["偏好1"],
    "pending_items": ["待办1"]
}
```

**存储**：摘要存入 conversation 对象的 `context_summary` 字段。

### P1-4 调度轮次优化

**三管齐下**：
1. 每轮注入当前可用工具列表到 system_prompt
2. 截断时优先保留含 tool_call 的消息
3. 连续2轮无 tool_call 时注入"工具使用提醒"

### P1-5 TODO 前端可视化

**后端**：新增 `GET /api/conversations/{conv_id}/todos` 返回所有 TODO 清单。

**前端**：新建 `TodoPanel.vue`，在 ChatWindow 右侧或顶部展示看板。

### P1-6 文件版本控制

**备份策略**：write_file/edit_file 执行前，将旧文件复制到 `.agent/file_history/{文件名}/{时间戳}`。

**新工具**：
- `file_diff` — 对比两个版本
- `file_rollback` — 回退到指定版本
- `list_file_versions` — 列出文件版本历史

### P1-7 智能体性能追踪

**数据结构**：
```python
"stats": {
    "total_tasks": 0,
    "completed_tasks": 0,
    "failed_tasks": 0,
    "avg_duration_sec": 0,
    "last_active_at": null
}
```

**更新时机**：`submit_task_result` 成功时 completed_tasks++，失败时 failed_tasks++。

## 四、实施顺序

1. P0-1 → P0-2 → P0-3（记忆体系，有依赖关系）
2. P1-4（调度优化，独立）
3. P1-5 → P1-6 → P1-7（功能补全，互相独立）
4. P2-8（消息拆分，独立）
5. 测试验证
