# 团队蓝图 Schema 规范

> 版本：1.0  
> 日期：2026-07-19  
> 状态：✅ 已实现 v1.0（2026-07-21 更新）

## 概述

团队蓝图（Team Blueprint）是元团队的核心产出，也是连接元团队和项目团队的桥梁。采用 JSON 格式，便于程序解析和人工编辑。

## Schema 定义

### 顶层结构

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `blueprint_version` | string | 是 | 蓝图版本，当前固定 `"1.0"` |
| `team_name` | string | 是 | 团队名称，简洁明确 |
| `description` | string | 否 | 团队适用场景的一句话描述 |
| `created_by` | string | 否 | 创建者，元团队产出时固定 `"元团队"` |
| `created_at` | string | 否 | ISO 8601 时间戳 |
| `members` | array | 是 | 成员清单，至少 2 个成员 |
| `tasks` | array | 否 | 任务清单，含依赖关系 |
| `recommended_config` | object | 否 | 推荐模型配置 |
| `execution_tips` | string | 否 | 执行建议 |

### members[] 结构

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `name` | string | 是 | 成员名称，中文 2-6 字 |
| `role` | string | 是 | 角色定位，一句话 |
| `system_prompt` | string | 是 | 完整系统提示词，200-500 字 |
| `collaboration` | object | 是 | 协作模式定义 |

### collaboration 结构

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `type` | string | 是 | 协作模式：`star` / `direct` / `parallel` |
| `report_to` | string | star 模式必填 | 汇报对象，固定 `"main"` |
| `with` | string | direct 模式必填 | 协作对象的 name |

### 协作模式说明

| 模式 | 含义 | 适用场景 |
|------|------|----------|
| `star` | 星型协作，向主智能体汇报 | 职责清晰、各干各的场景 |
| `direct` | 直接协作，成员之间 @对方 | 需要紧密配合的角色对 |
| `parallel` | 并行协作，多个成员同时独立工作 | 同类工作分给多人 |

### tasks[] 结构

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `name` | string | 是 | 任务名称 |
| `assignee` | string | 是 | 负责成员的 name |
| `depends_on` | array | 否 | 依赖的前置任务 name 列表，空数组表示无依赖 |
| `description` | string | 否 | 任务描述 |

### recommended_config 结构

| 字段 | 类型 | 说明 |
|------|------|------|
| `context_window` | integer | 推荐上下文窗口，如 131072 |
| `enable_thinking` | boolean | 是否开启思考模式 |
| `max_tokens` | integer | 推荐最大输出长度 |

## 校验规则

后端 `validate_blueprint` 函数会检查以下规则：

1. **JSON 合法性**：必须是合法 JSON
2. **顶层必填字段**：`blueprint_version`、`team_name`、`members`
3. **成员必填字段**：每个 member 必须有 `name`、`role`、`system_prompt`、`collaboration`
4. **协作类型合法性**：`collaboration.type` 只能是 `star`、`direct`、`parallel`
5. **star 模式**：必须有 `report_to`
6. **direct 模式**：必须有 `with`，且 `with` 指向的成员在 members 里存在
7. **任务依赖无环**：`tasks[].depends_on` 不能形成循环依赖
8. **任务 assignee 存在**：每个任务的 `assignee` 必须在 members 里存在
9. **成员数量**：至少 2 个成员，不超过 8 个

## 版本管理

- `blueprint_version` 用于后续版本兼容
- 当前版本 `1.0`，后续如有破坏性变更升级到 `2.0`
- `apply-blueprint` 接口会根据 version 选择对应的解析逻辑
