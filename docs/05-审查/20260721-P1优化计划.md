# P1 优化计划

> 日期：2026-07-21 ｜ 对应版本：v0.4.0 ｜ 状态：待执行

## 背景

P0 全部修复后（12 项），还剩 24 个 P1 问题。本计划聚焦其中影响**安全性和数据一致性**的 14 项，按 4 个阶段推进，每阶段完成后验证。

## 阶段划分

### 阶段 1：安全加固（4 项）

| 编号 | 问题 | 修复方式 | 风险 |
|---|---|---|---|
| P1-S02 | tasks 目录权限过大，子智能体能读写其他任务 | allowed_paths 只授权当前 task_dir_abs，移除 tasks_abs 前缀 | 中 |
| P1-S03 | 文件上传无大小/类型限制 | 添加 10MB 限制 + 危险扩展名黑名单（.exe/.bat/.ps1/.sh/.cmd/.com/.vbs） | 低 |
| P1-S04 | 幻觉检测不覆盖子智能体 | 在子智能体回复处理后也执行 task_id 引用校验 | 中 |
| BE-补 | write_file/edit_file 缺回读校验 | 写入后回读文件验证字节数，参照 move_file 模式 | 低 |

### 阶段 2：数据一致性（4 项）

| 编号 | 问题 | 修复方式 | 风险 |
|---|---|---|---|
| P1-D02/D03 | create_conversation/create_team 遗漏字段 | 复制模板时补全 enabled_skills=[] 和 can_invoke_sub_agent=False | 低 |
| P1-D05 | load_agents 迁移逻辑未补全字段 | 迁移逻辑中补全两个新字段 | 低 |
| P1-D06 | apply_blueprint 创建 main 遗漏字段 | 创建 main 时补全字段 | 低 |
| P1-F05 | populateEditForm 缺三个字段 | 编辑表单补全 model_config_id/enabled_skills/can_invoke_sub_agent | 低 |

### 阶段 3：功能完善（4 项）

| 编号 | 问题 | 修复方式 | 风险 |
|---|---|---|---|
| BE-补 | invoke_sub_agent 缺 thinking_override | 调用时传入参数 | 低 |
| BE-补 | extra_rounds 未用隔离上下文 | 改用 build_isolated_context | 中 |
| P1-M02 | 蓝图导入时无校验 | 导入前调用 validate_blueprint_data | 低 |
| BE-补 | load_json 异常处理过于宽泛 | 区分 JSONDecodeError 和其他异常，损坏时记日志 | 低 |

### 阶段 4：前端清理（2 项）

| 编号 | 问题 | 修复方式 | 风险 |
|---|---|---|---|
| P1-F01 | ApiConfigDialog.vue 死代码 | 直接删除 | 低 |
| P1-F04 | SSE data 解析用 .trim() | 改为只去掉 "data:" 后的 1 个空格 | 低 |

## 不在本批次的问题（后续迭代）

- P1-F02/F03：文件图标和工具名称映射重复（需抽取公共模块，工作量大）
- P1-M01/M03：元团队评审专家和异常事件检测（需深入理解业务逻辑）
- fast 模式、融合采纳分析（需产品决策）
- 所有 P2 架构优化

## 验证方式

每阶段完成后：
1. 后端 `python -c "import main"` 验证导入
2. 前端 `npm run build` 验证构建
3. 最后启动后端，运行 API 测试脚本验证关键修复
