## 轮次：v0.5 核心优化 — 记忆体系 + 功能补全（P0+P1）

**做的事**：
按能力评估报告逐项推进 8 个优化方向，后端 + 前端全栈实现：

### 后端改动（`backend/main.py`）

1. **P0-1 智能体记忆系统**：
   - 新增 6 个记忆工具函数：`format_memory_for_prompt`、`add_memory_to_agent`、`extract_memory_from_conversation`、`trigger_memory_extraction`、`sync_memory_to_team`、`inject_team_memory_to_agent`
   - 修改 `build_conversation_history`：新增 `agent` 参数，自动注入 `memory_log` 到系统提示词
   - 修改 `submit_task_result`：任务完成后触发 LLM 记忆提取（静默失败）
   - 新增 3 个 API 端点：GET/DELETE/POST `/api/conversations/{id}/agents/{agent_id}/memory`

2. **P0-2 跨项目经验迁移**：
   - `sync_memory_to_team`：项目级记忆超 10 条时自动同步到 teams.json 团队级 `shared_memory`
   - `inject_team_memory_to_agent`：新项目创建时自动继承团队级共享记忆
   - 修改 `create_conversation`：创建项目后调用 `inject_team_memory_to_agent`

3. **P0-3 对话摘要压缩**：
   - 修改 `build_conversation_history` 截断逻辑：被截断的消息生成摘要（用户消息保留 200 字、智能体消息保留 100 字）注入 system 消息

4. **P1-4 调度轮次优化**：
   - 截断策略优先保留含工具调用关键词的消息（防止多轮后工具调用退化）
   - CORE_RULES 新增第 9 条"工具使用防退化"规则

5. **P1-5 TODO API**：新增 `GET /api/conversations/{id}/todos` 端点

6. **P1-6 文件版本控制**：
   - 新增 `_backup_file_version` 函数，`write_file`/`edit_file` 执行前自动备份到 `.agent/file_history/`（保留 10 版）
   - 新增 `list_file_versions`、`file_rollback` 两个工具定义 + 执行逻辑 + MAIN_ONLY_TOOLS 登记

7. **P1-7 性能追踪**：
   - `submit_task_result` 成功后更新 agent 的 `stats` 字段（completed_tasks、avg_duration_sec、last_active_at）
   - 新增 `GET /api/conversations/{id}/agents/stats` API 端点

### 前端改动
