# 多智能体团队软件 - 技术规划

## 一、项目目标（要做什么）

做一个网页版的多智能体团队协作软件，类似"AI 微信群"：
- 你（主智能体）在群里给不同的 AI 智能体安排活儿
- 每个智能体有自己的角色和能干的事
- 每个智能体只能看到/改动你允许它碰的文件夹
- 可以自由切换用哪家的大模型（本地或云端）

## 二、核心功能清单

| 编号 | 功能 | 说明 | 优先级 |
|------|------|------|--------|
| F1 | 聊天主界面 | 类似微信，左侧智能体列表，右侧聊天窗口 | 必做 |
| F2 | 智能体角色配置 | 给每个智能体起名、设性格、设职责 | 必做 |
| F3 | 文件夹权限 | 设置每个智能体能读/能写哪些文件夹 | 必做 |
| F4 | 主智能体调度 | 主智能体能把任务派给团队里的其他智能体 | 必做 |
| F5 | API 配置面板 | 填 API 地址和密钥，能选本地或云端模型 | 必做 |
| F6 | 智能体新建/删除 | 团队成员管理 | 必做 |
| F7 | 聊天记录保存 | 关掉软件再打开还能看到之前的对话 | 可选 |
| F8 | 多种模型切换 | 一个智能体可以换不同的模型 | 可选 |

## 三、技术方案（用什么做）

**整体思路**：一个网页应用，分两部分
- **用户看到的部分（前端）**：聊天界面、配置面板
- **用户看不到的部分（后端）**：调度智能体、调 API、管文件权限

**技术选型**：
| 部分 | 用什么 | 为什么选它 |
|------|--------|-----------|
| 前端界面 | Vue 3 + Vite | 组件化好维护，响应式自动更新，用户已熟悉 |
| UI 组件库 | Element Plus | 现成组件多，聊天框/表单/弹窗都现成，省事 |
| 后端服务 | Python + FastAPI | 异步支持好（多智能体并发），自动生成 API 文档，AI 项目主流 |
| 调用大模型 | openai 库（兼容几乎所有 API） | 一套代码调所有模型，包括本地 |
| 前后端通信 | axios | Vue 生态标配，简单可靠 |
| 数据存储 | JSON 文件 | 不用装数据库，简单够用 |

**为什么要后端**：
- 网页不能直接读写你电脑的文件夹（浏览器安全限制），必须后端中转
- API 密钥不能放前端（会泄露），必须放后端
- 多智能体调度逻辑在后端跑更稳定

**为什么选 FastAPI 而不是 Flask**：
- 多智能体要同时调多个 API，FastAPI 异步原生支持，不卡顿
- 自动生成 API 文档，调试方便（浏览器打开 /docs 就能看）
- AI 项目主流选择，生态好

**依赖清单**：
- 前端：vue、vue-router、pinia（状态管理）、element-plus、axios（5 个，npm 一键装）
- 后端：fastapi、uvicorn（服务器）、openai、python-dotenv（4 个，pip 一键装）

## 四、界面长什么样

```
┌─────────────┬──────────────────────────────┐
│ 智能体列表    │  聊天窗口                     │
│             │                              │

│             │  ┌────────────────────────┐  │
│ ─────────  │  │ 输入消息...        [发送]│  │

│ ⚙️ API设置  │                              │
```

## 五、实现步骤（分 4 步走）

**第 1 步：搭骨架**
- 做出能显示的网页界面（聊天框 + 智能体列表）
- 后端能启动，前端能连上
- 暂时不接 AI，先让界面能"假对话"

**第 2 步：接通 AI**
- 做 API 配置面板
- 让一个智能体能真正调用大模型回复
- 主界面显示真实回复

**第 3 步：多智能体调度**
- 主智能体能把消息转给其他智能体
- 其他智能体回复后汇总显示
- 角色配置生效（不同智能体有不同性格）

**第 4 步：加权限和存档**
- 文件夹权限配置（每个智能体能读写哪些目录）
- 聊天记录保存到 JSON 文件
- 新建/删除智能体

## 六、第一版不做的功能（边界）

- ❌ 不做手机端（先 Web，跑通再说）
- ❌ 不做用户登录（个人用，不需要账号系统）
- ❌ 不做云端部署（先本地跑）
- ❌ 不做多用户协作（单人使用）
- ❌ 不做复杂的权限继承（白名单模式即可）

## 七、风险点

1. **主智能体调度逻辑**：怎么让主智能体"知道"该派给谁？第一版用最简单方式：主智能体在回复里提到某个智能体名字，就自动转发。
2. **文件夹权限安全**：网页应用直接读写电脑文件夹有风险，第一版先限制在项目目录内。
3. **API 兼容性**：不同模型 API 格式不一样，用 openai 库统一（绝大多数都兼容 openai 格式）。
